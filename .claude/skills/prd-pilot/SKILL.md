# PRD Pilot

## When to use this skill

Use this skill when the user wants to turn a vague product idea into a structured requirement, a Chinese PRD draft, a demo-ready HTML prototype, or a targeted iteration plan.

## Workflow

1. Structure the raw idea into a shared Requirement Spec.
2. Generate the PRD and/or demo from the same Requirement Spec.
3. Run a consistency check between the Requirement Spec, PRD, demo HTML, and prototype outline.
4. Apply targeted iteration instead of rewriting everything.

## Output expectations

- Keep the workflow PM-focused.
- Prefer concise, reviewable outputs.
- Keep PRD and demo naming aligned.
- Preserve the shared Requirement Spec as the source of truth.

## Do not

- Do not turn this into a generic agent planner.
- Do not introduce prefab marketplaces or PocketFlow-style orchestration.
- Do not drift away from the PM validation workflow.
