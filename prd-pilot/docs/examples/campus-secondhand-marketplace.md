# 校园二手交易平台

## 1. 输入

- 产品名称：校园二手交易平台
- 产品类型：Web 工具 / 校园社区产品
- 核心目标：让学生快速发布、浏览和交易二手商品，并在评审时能看到完整的交易流程。
- 重点诉求：列表清晰、发布简单、聊天/下单流程明确、带状态变化。

## 2. Requirement Spec

- 目标用户：在校学生、二手商品卖家、买家、社团管理员
- 用户痛点：
  - 商品信息分散，检索效率低
  - 发布流程繁琐，商品状态难以管理
  - 交易后反馈与纠纷入口不清楚
- 核心场景：
  - 浏览首页推荐商品
  - 发布闲置商品
  - 查看商品详情并联系卖家
  - 创建订单并跟踪状态
  - 交易完成后评价或反馈问题
- 关键功能：
  - 商品发布与编辑
  - 商品列表和筛选
  - 订单状态流转
  - 消息和反馈入口
- 主要页面：
  - 首页 / 商品列表
  - 商品详情页
  - 发布商品页
  - 我的订单页
  - 消息 / 反馈页
- 原型风格：后台管理台 + 校园社区感
- 成功标准：
  - 需求评审时能快速看懂交易主线
  - Demo 中至少有“浏览 -> 详情 -> 发布 -> 订单”完整闭环

## 3. PRD 要点

- 首页增加商品卡片、热卖标签和筛选器
- 商品详情页展示图片、价格、成色、卖家信息和联系按钮
- 发布商品页支持图片、标题、描述、价格、分类和成色输入
- 订单页展示待支付、待发货、待确认、已完成等状态
- 反馈页支持交易问题和举报入口

## 5. 一次修改结果

反馈示例：

> 详情页的信息太散了，希望把“联系卖家”“收藏”“下单”收敛到右侧操作区，并增加订单状态提示。

修改结果：

- 把核心操作集中到详情页右栏
- 增加“交易状态”提示条，明确当前可执行动作
- 在订单页补充状态标签，避免用户不知道下一步做什么
- 让首页的进入路径更短，从列表页直接进入交易链路
